## Détail des changements

### PR #29 - Linux backend-libs path (BUG-051)
- Sur Linux .deb, `resource_dir()` renvoyait `/usr/lib/THERESE/` mais les fichiers backend étaient dans `binaries/backend-libs/`
- Fix : `.join("backend-libs")` → `.join("binaries").join("backend-libs")` dans `lib.rs`
- Testé sur DreamQuest Ludo (Ubuntu 24.04)

### PR #30 - Linux runtime hook permissions (BUG-052)
- `runtime_hook_templates.py` tentait `os.makedirs` dans `/usr/lib/THERESE/` (lecture seule root)
- Fix : catch `OSError` autour des `os.makedirs`
- Testé sur DreamQuest Ludo (Ubuntu 24.04)

### PR #31 - Email refresh diagnostic (BUG-066)
- Le message "Échec du rafraîchissement" masquait la cause réelle
- Fix : affichage de l'erreur réelle dans le bandeau (NETWORK/AUTH/API) + logs console structurés

### PR #32 - Commandes utilisateur (BUG-067)
- Commandes créées via RFC invisibles (catégorie "personnaliser" non affichée)
- Slash commands ne faisaient que mettre `/{nom} ` au lieu d'injecter le prompt_template
- Fix : catégorie par défaut "production", injection prompt_template, ajout suppression/déplacement de commandes
- 10 fichiers modifiés (6 frontend + 4 backend)

### PR #33 - Board historique + UI (BUG-068)
- Clic "Historique" du Board = panneau vide (erreur 500, colonne `mode` manquante)
- Fix : auto-migration `ALTER TABLE` dans `database.py`
- UI : suppression dégradés boutons d'action → couleur unie accent-cyan
