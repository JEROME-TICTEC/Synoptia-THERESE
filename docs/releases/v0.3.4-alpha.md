## Problèmes découverts et corrigés pendant la release

### Workflow release cassé (BUG-044 collatéral)
La PR #27 avait introduit un heredoc bash dans `release.yml` dont le marqueur `EOF` en colonne 0 cassait le parsing YAML de GitHub Actions. Le fichier entier devenait invalide :
- Le `workflow_dispatch` trigger n'était plus reconnu
- Les tag push ne déclenchaient plus le build
- Le nom du workflow apparaissait comme le chemin fichier au lieu de "Release THÉRÈSE"

**Fix** : remplacement du heredoc par `printf` (commit f9a9d6c).

### Glob resources Tauri cassé
Le glob `binaries/backend-libs/**` dans `tauri.conf.json` ne matchait aucun fichier sur aucune plateforme :
- Sur macOS/Windows : le placeholder `.gitkeep` n'était pas matché par `**` (fichiers cachés)
- Sur Linux : même avec 1.1G de fichiers, `**` dans le glob Rust ne matche que des **répertoires**, pas des fichiers

**Fix** :
1. `tauri.conf.json` : `resources: []` (vide par défaut)
2. Nouveau `tauri.linux.conf.json` avec globs `backend-libs/*` + `backend-libs/_internal/**/*`
3. Workflow : `--config` override uniquement sur Linux (commit 6743124 + a8987d8)

### uv.lock désynchronisé
Le lockfile Python était resté à la version 0.3.0 (non mis à jour lors du bump sur DreamQuest).
**Fix** : `uv lock` puis commit (1d26f94).
