# Release v0.8.0-alpha

> Date : 25 mars 2026

## Résumé

Release majeure avec 3 nouvelles features et 4 corrections de bugs.
Session marathon : triage Discord, jury v0.8, implémentation et release en une journée.

## Changements

### Nouvelles fonctionnalités

**1. Détection auto de skills + syntaxe {{action}} (BUG-093)**
- L'IA détecte automatiquement quand l'utilisateur demande un document (PPTX, DOCX, XLSX, etc.)
- Syntaxe explicite `{{action: pptx-pro}}` pour forcer un skill
- Exécution automatique du skill après le streaming → fichier téléchargeable
- Fichiers : `intent_detector.py`, `chat.py`, `schemas.py`, `ChatInput.tsx`, `chat.ts`

**2. Workspace tools pour l'IA (email + calendrier)**
- 5 tools LLM : read_emails, send_email, search_emails, list_calendar_events, create_calendar_event
- L'IA peut lire/envoyer des mails et gérer le calendrier en conversationnel
- Fichier : `workspace_tools.py`, `chat.py`

**3. Module devis/avoir dans la facturation**
- Sélecteur type document (Devis/Facture/Avoir) dans InvoiceForm
- Filtres par type + badges dans InvoicesPanel
- Endpoint POST /api/invoices/{id}/convert (copie avec nouveau type/numéro)
- Le backend supportait déjà document_type, seul le frontend manquait

**4. Import ICS (calendrier) et VCard (contacts)**
- Parser ICS (icalendar) et VCF (vobject) dans import_service.py
- Endpoint POST /api/calendar/import-ics (dédoublonnage par UID)
- Endpoint POST /api/crm/import/vcf (dédoublonnage par email)
- Boutons Upload dans CalendarPanel et CRMPanel
- Dépendance ajoutée : vobject 0.9.9

### Corrections de bugs

**5. PDF facture : sécurisation et chemin**
- Fix .get() sur dataclass UserProfile (bug pré-existant trouvé par audit)
- PDF généré dans le dossier de travail configuré (plus hardcodé ~/.therese/invoices)
- Décimal pavé numérique accepté (point ET virgule)
- Tests ajoutés

**6. Agents partagent les clés API**
- Le cache de clés API n'était pas chargé au démarrage
- Les agents de l'Atelier utilisent maintenant les clés configurées dans Paramètres > IA
- Reload du cache après MAJ de clé

**7. OpenRouter : liste dynamique des modèles**
- Remplacé la liste hardcodée (6 modèles fixes) par un fetch dynamique depuis l'API OpenRouter
- Fallback sur la liste statique si pas de clé ou erreur réseau
- MAJ modèles agents : Nemotron obsolète remplacé par Claude/GPT/Gemini/Llama
