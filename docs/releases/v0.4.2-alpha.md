## Contenu

### F-17 : Refonte Board de Décision + Mode Souverain Ollama

**Backend** :
- Ajout `BoardMode` enum (CLOUD / SOVEREIGN) dans `models/board.py`
- Champs `mode` et `ollama_models` ajoutés à `BoardRequest`
- Champ `mode` ajouté à `BoardDecisionDB` (entities.py)
- Service `board.py` : chemin séquentiel pour mode souverain (boucle for au lieu d'asyncio.gather), pas de recherche web, Ollama par conseiller
- `llm.py` : paramètre `model_override` ajouté à `get_llm_service_for_provider()`

**Frontend** :
- `ModeSelector.tsx` (NOUVEAU) : toggle Cloud/Souverain avec animation Framer Motion `layoutId`
- `AdvisorArcLayout.tsx` (NOUVEAU) : 5 conseillers en arc de cercle (angles -50° à +50°), fallback grille responsive, select modèle Ollama par conseiller
- `BoardPanel.tsx` : intégration ModeSelector + AdvisorArcLayout, détection Ollama auto, bouton premium gradient, bouton PDF désactivé
- `AdvisorCard.tsx` : état "waiting" (opacity 40%), avatar 56px avec outline, glow loading
- `DeliberationView.tsx` : barre de progression par conseiller, bannière mode souverain, conseillers en attente grisés
- `board.ts` : types TypeScript mis à jour (BoardMode, mode, ollama_models)
- `index.ts` : exports ModeSelector + AdvisorArcLayout

### BUG-056 : Indicateur de chargement image

- `chatStore.ts` : `updateMessage` accepte un 3e paramètre `meta?: Partial<Message>`
- `CommandExecutor.tsx` : message loading `isStreaming: true` avant l'appel API, remplacement par `updateMessage` après
- `GuidedPrompts.tsx` : même pattern

### F-16 : Import credentials.json Google OAuth

- `CredentialsStep.tsx` : input file + bouton Upload, parse `installed.client_id` / `web.client_id`
- `CRMSyncPanel.tsx` : même pattern dans le formulaire CRM

### PR fermée

- **#28** (Zézette) : BUG-044b Linux backend-libs - fermée car doublon (déjà fixé v0.3.6)
