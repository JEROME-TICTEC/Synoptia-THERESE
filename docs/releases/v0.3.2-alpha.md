## Résumé fonctionnel

### BUG-042 : Documents DOCX tronqués (Opus/Mistral)
- **Problème** : Le `max_tokens` à 8192 était insuffisant pour les modèles verbeux (Opus, Mistral) qui génèrent du code Python long pour créer les documents Office.
- **Fix** : `max_tokens` porté à 16384 dans `skills.py`.

### BUG-043 : Documents vides (juste le titre)
- **Problème** : Le LLM générait un code Python minimaliste (ex: `doc = Document(); doc.save(...)`) qui passait la validation syntaxique mais produisait un document vide.
- **Fix** : Ajout de `_validate_document_content()` dans `code_executor.py` qui vérifie le contenu réel du fichier généré (paragraphes pour DOCX, slides pour PPTX, lignes pour XLSX). Si quasi vide, fallback vers le parser legacy. En complément, retry automatique en mode Markdown dans `skills.py` si le document reste vide après le fallback.

### BUG-044 : Fichiers joints ignorés dans le chat
- **Problème** : Le frontend capturait les fichiers glissés-déposés mais ne transmettait pas les chemins (`file_paths`) au backend. Le backend ne les traitait donc jamais.
- **Fix** : Ajout du champ `file_paths` dans `ChatRequest` (schema), propagation dans le streaming et le non-streaming, traitement via `_get_file_context()` existant (avec validation sécurité `validate_file_path`).
