# Release v0.6.3-alpha

**Date** : 13 mars 2026
**Mode** : Fast-track (changements directs, pas de PRs)
**Version** : 0.6.2 → 0.6.3

## Changements

### Fix critique - Onboarding / perte de données
- **Backend** (`routers/config.py`) : `GET /onboarding-complete` détecte maintenant une DB existante (conversations/contacts) et crée le flag rétroactivement si absent
- **Frontend** (`App.tsx`) : fallback localStorage supprimé - si l'API échoue 5 fois, on ne montre PAS l'onboarding (safe default)
- **Contexte** : la v0.6.2 a causé la perte de 42 contacts, 32 conversations et 105 messages lors d'une reinstall

### RGPD - DELETE /api/data/all complet
- 12 tables manquantes ajoutées : InvoiceLine, Invoice, CalendarEvent, Calendar, EmailLabel, EmailMessage, EmailAccount, Task, Deliverable, Activity, PromptTemplate, AgentTask, AgentMessage, CodeChange
- Purge Qdrant (collection vectorielle) ajoutée
- Ordre de suppression respecte les FK

### Performance - Singleton SQLite
- 4 appels `create_engine()` ad-hoc dans `llm.py` remplacés par `get_sync_connection()` (singleton `database.py`)
- Réutilise les PRAGMAs WAL/cache/mmap au lieu de créer des engines jetables
- Nouvelle fonction `get_sync_connection()` dans `database.py`

### Qualité code - Ruff B+C90+SIM
- Règles `B`, `C90`, `SIM` activées dans `pyproject.toml`
- B905 corrigé (zip strict=True), B007 corrigés, SIM auto-fixés
- Ignores ciblés : B008 (FastAPI Depends), C901 (complexité progressive), B904 (raise from progressive)

### Atelier - Nouveaux modèles LLM
- **Backend** (`services/agents/config.py`) : Nemotron 3 Super 120B + Nano 30B (OpenRouter), Mistral Large 3 fixé ajoutés
- **Backend** (`services/agents/runtime.py`) : support modèles OpenRouter (format `provider/model` avec `/`)
- **Frontend** (`LLMTab.tsx`) : Gemini Flash Lite, Mistral Large 3 fixé, Nemotron 3 Super + Nano ajoutés

### DevOps - Script bump-version.sh
- `scripts/bump-version.sh` : met à jour les 6 fichiers de version en une commande
- Validation semver + vérification post-bump
