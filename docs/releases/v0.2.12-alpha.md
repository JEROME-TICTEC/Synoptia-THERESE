## Détail des correctifs

### PR #19 - Batch v0.2.12 (Zézette)

- **BUG-031** : THÉRÈSE chargeait les 50 premiers messages au lieu des 50 derniers dans les longues conversations. Fix : `.order_by(created_at.desc()).limit(50)` puis `reversed()`.
- **CRM PUT→PATCH** : Les modifications de contacts et projets CRM étaient silencieusement perdues (405 côté serveur). Fix : `method: 'PATCH'` dans `memory.ts`.
- **BUG-048** : Skills Office tronqués avec Ollama (défaut 128 tokens). Fix : `num_predict` et `num_ctx` transmis dans les options.
- **asyncio Python 3.13** : 11x `get_event_loop()` → `get_running_loop()` dans `imap_smtp_provider.py`.
- **Refresh token Google** : Sauvegarde du nouveau refresh_token si Google en retourne un lors du renouvellement.
- **BUG-032** : Extensions fichiers enrichies dans le dialog d'upload.
- **GET /files/content** : `extract_text()` remplace le placeholder TODO.

### PR #20 - Fix audit : extensions mortes

Extensions `.xls`, `.ods`, `.pptx`, `.ppt`, `.odt`, `.doc` retirées du filtre ChatInput car non supportées par `extract_text()`. Le filtre ne liste désormais que les formats réellement traités : `txt, md, pdf, docx, xlsx, json, csv`.

### PR #21 - Fix audit : tri déterministe

Ajout de `Message.id.desc()` comme clé de tri secondaire dans le chargement d'historique chat, pour éviter un ordre non déterministe quand deux messages ont le même timestamp en SQLite.

### PR #23 - Fix audit : caldav asyncio

9x `get_event_loop()` → `get_running_loop()` dans `caldav_provider.py`. Même pattern que le fix de `imap_smtp_provider.py`, appliqué au module calendrier CalDAV.
